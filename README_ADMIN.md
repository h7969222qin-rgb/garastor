# GARASTOR Admin Dashboard — Setup & Usage

This is the visual dashboard that lets you (or the client) publish products and
vlog articles to the live site **without touching code or the command line**.
It commits to GitHub via the API; Cloudflare Pages auto-deploys.

- **URL**: `https://garastor1.pages.dev/admin.html`  (old `/publisher` redirects here)
- **Zero payment card needed**: uses only GitHub free API + CF Pages free deploy.
  No R2 / KV / Functions / bind-card.

---

## One-time setup (you do this once)

### 1. Create a GitHub Personal Access Token

Fine-grained token (recommended):
1. GitHub → Settings → Developer settings → Personal access tokens → Fine-grained → Generate.
2. Resource owner: your account.
3. Repository access: **Only select repositories** → `garastro`.
4. Permissions → Repository permissions:
   - **Contents**: Read and write
   - **Metadata**: Read-only (auto-required)
5. Generate → copy the token (`github_pat_...`).

(Classic token also works: check the `repo` scope.)

### 2. Generate `token.enc`

1. Open `https://garastor1.pages.dev/admin.html` (or `dist/admin.html` locally).
2. Click the **Setup** tab.
3. Paste the GitHub token + choose your dashboard password.
4. Click **Generate & Download token.enc** → downloads `token.enc`.

This file is the token **encrypted with your password** (AES-256-GCM via PBKDF2).
Safe to put in the repo — without the password it's useless.

### 3. Commit `token.enc`

Put `token.enc` in **both** the repo root and `dist/`, then push. After CF deploys,
the dashboard login works.

```bash
# token.enc ends up in both root and dist after you place it
git add token.enc dist/token.enc admin.html dist/admin.html products.html dist/products.html publisher.html dist/publisher.html
git commit -m "Add admin dashboard (GitHub API publishing)"
git push
```

> The dashboard password is whatever you typed in Step 2 — it is never stored
> anywhere; it only decrypts `token.enc` in the browser.

## Daily use (publish products / vlog)

### Publish a product
1. Open `garastor1.pages.dev/admin.html` → enter password → Unlock.
2. **Product** tab:
   - Category (Herringbone / Chevron / Strip Plank)
   - Product Name (English, no spaces — becomes the folder name, e.g. `Capri`)
   - Chinese Name (optional)
   - Drag in photos (first photo = `1.jpg`)
3. **★ Publish Product to Live**.
4. Wait ~10s → done. Site auto-redeploys in ~1 min. Refresh `/products`.

What it commits in one atomic commit:
- `images/products/{cat}/{name}/1.jpg … N.jpg`
- updated `products-data.json`
- rewritten `products.html` (new product card inserted)

### Publish a vlog / article
1. **Vlog / Article** tab:
   - Title, Date, Tag, Slug (English URL name, e.g. `color-trends-2026`)
   - Drag a **cover photo**
   - Excerpt (one-line summary)
   - Body: paragraphs separated by blank lines; type `[IMG]` on its own line to
     insert a body photo there
   - Drag **body photos** (optional, matched to `[IMG]` placeholders in order)
2. **★ Publish Vlog to Live**.

What it commits:
- `images/vlog/{slug}.jpg` (cover) + `images/vlog/{slug}-2.jpg …` (body images)
- new article page `journal/{slug}.html`
- updated `journal.html` (new article prepended to the list)

## Troubleshooting

| Symptom | Fix |
|---|---|
| Login: "Wrong password, or token.enc missing/corrupt" | Wrong password, or `token.enc` not committed yet. |
| Publish: 401 / "Bad credentials" | Token expired or revoked — generate a new one, re-encrypt. |
| Publish: 403 / "Resource not accessible" | Token lacks Contents write permission, or wrong repo. |
| Publish: 404 on raw fetch | Branch isn't `main`, or `products-data.json` / `journal.html` not in repo root. |
| Product shows but image broken | Photo folder name doesn't match Product Name — check case/spacing. |
| Nothing happens after publish | Check the log box (black) for the GitHub API error. |

## Security model

- The GitHub token is **encrypted at rest** in `token.enc`. Anyone reading the
  file gets only ciphertext.
- The dashboard password lives only in your head + the browser session
  (not stored server-side; the dashboard is static HTML).
- The token is **repo-scoped**: fine-grained tokens can be limited to `garastro`
  with only Contents write — worst case a leak lets someone edit the website.
- To rotate: generate a new token, re-run Setup, replace `token.enc`, push.

## Files

| File | Role |
|---|---|
| `admin.html` (+ `dist/admin.html`) | The dashboard (login, Product tab, Vlog tab, Setup tab) |
| `token.enc` (+ `dist/token.enc`) | Encrypted GitHub token (you generate + commit) |
| `publisher.html` | Redirects to `admin.html` |
| `products.html` | Regenerated by the Product tab on each publish |
| `journal.html` / `journal/{slug}.html` | Regenerated / created by the Vlog tab |
